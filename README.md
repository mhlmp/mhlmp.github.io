## 🎯 חזון ומטרת המערכת
המערכת **MH Link Manager Pro** פותחה כדי להחליף את מערכות ה"מועדפים" (Bookmarks) המסורתיות והמיושנות של הדפדפנים. האפליקציה מתפקדת כ"מוח שני" עבור קישורים, קטעי קוד וטקסטים חשובים.

במקום לשמור רק כותרת וכתובת אינטרנט (URL), האפליקציה ניגשת באופן אקטיבי לאתר הנשמר. המערכת עוקפת חסימות בסיסיות, קוראת את התוכן שלו, ומעבירה אותו לעיבוד במנועי בינה מלאכותית מתקדמים (LLMs). התוצאה היא מאגר נתונים עשיר שבו כל קישור מלווה בתקציר ענייני בעברית, תגיות חכמות לחיפוש מהיר, וחיווי סטטוס מדויק (האם האתר עדיין פעיל, חונה או שבור).

## ✨ יכולות ויחודיות המערכת

### 🤖 אינטגרציית בינה מלאכותית (AI) וסריקת עומק
* **מנוע סריקה מרובה ספקים:** תמיכה מובנית ב-API של **Google Gemini** (ברירת מחדל), **Groq** (עבור Llama 3 סופר-מהיר), ו-**OpenRouter**. המערכת מאפשרת מעבר מהיר בין המנועים במקרה של ניצול מכסות (Rate Limits).
* **עקיפת הגנות מורכבות ופענוח טלגרם (Anti-Bot Bypass):** המערכת מזהה קישורים ומנסה לשלוף תוכן דרך מערך שרתי Proxy כדי לעקוף חסימות (כמו Cloudflare). כולל מנגנון ייעודי וחכם לניתוח קישורי **Telegram** המזהה קישורי הזמנה סגורים או פגי-תוקף המוסתרים מאחורי דפי "Join Group" גנריים.
* **אופטימיזציה לחיפוש (SEO אישי):** יצירת תקצירים בעברית תקנית תוך שמירה מכוונת של שמות מותגים, מונחים טכניים וקוד בשפת המקור (אנגלית) ובתוך תגיות (Hashtags), כדי למקסם את יעילות החיפוש העתידי.
* **פענוח אוטומטי של סטטוס הרשת:** ה-AI מונחה לזהות ולהחזיר סטטוס מדויק לכל קישור – זיהוי אתרים פעילים (🟢), דומיינים למכירה/חניה (🟠), או אתרים שקרסו (🔴).
* **אוטומציה שקופה:** אפשרות להפעלת "סריקת רקע אוטומטית" כך שכל פריט חדש שנוסף עובר ניתוח AI ללא מגע יד אדם.

### ☁️ ארכיטקטורת ענן, סנכרון ועבודה ללא רשת (Offline)
* **אסטרטגיית עדיפות מקומית (Offline First):** האפליקציה בנויה על בסיס IndexedDB לאחסון מקומי ו-Service Workers לניהול מטמון. התוצאה: זמני טעינה אפסיים ויכולת קריאה, חיפוש ועריכה מלאה גם במצב טיסה (ללא אינטרנט כלל).
* **תור סנכרון מאובטח (Push Queue Architecture):** כדי למנוע דריסת נתונים (Race Conditions) בעבודה מקבילה (למשל סמארטפון ומחשב נייד יחד), פותח מנגנון תור אסינכרוני הדואג שפעולות שמירה ל-Firebase יבוצעו באופן טורי ומבוקר.
* **מנוע ייבוא וגיבוי חכם (Smart Merge):** בעלות מלאה על הנתונים (JSON & TXT). בעת ייבוא קבצי גיבוי, המערכת מפעילה מנגנון מיזוג חכם המזהה פריטים קיימים (לפי ID ייחודי) ומעדכן אותם דינמית מבלי ליצור כפילויות. הייצוא לטקסט כולל קידוד מיוחד לשמירה על ירידות השורה המקוריות של סיכומי ה-AI.

### 🎨 ממשק משתמש (UI/UX) וחוויית שימוש
* **בחירה מרובה תלוית-הקשר (Smart Multi-Select):** ממשק ייעודי המאפשר לבחור עשרות פריטים *מתוך מה שמוצג כרגע על המסך בלבד* (תוך התחשבות בסינון וחיפוש פעיל), לצורך העברת קטגוריה, מחיקה או **הפעלת סריקת AI מרובה וממוקדת לנבחרים**.
* **עיצוב זכוכית מודרני (Glassmorphism):** ממשק נקי, צף ויוקרתי, המגיב בזמן אמת וכולל תמיכה מלאה במעבר חלק בין מצב יום למצב לילה (Dark Mode).
* **מערכת התראות חכמה (Toasts):** חיוויים צפים דינמיים המעדכנים על התקדמות סריקות, שגיאות רשת או פעולות מערכת. אלמנטים על המסך (כמו כפתור גלילה מעלה) מזהים את ההתראות וזזים אוטומטית כדי לא להסתיר אותן.

### 🛡️ אבטחה, פרטיות ושליטה
* **הגנת חוקרי קוד (DevTools Protection):** מנגנון ייעודי לזיהוי וחסימה של פתיחת כלי המפתחים (Console/Inspect Element). המנגנון מגן על סביבת העבודה מפני משתמשים לא מורשים ונסגר אוטומטית רק עבור כתובת אימייל מוגדרת מראש של מנהל המערכת.
* **ניהול אשפה בטוח:** סל מחזור (Trash) מובנה ששומר פריטים שנמחקו (Soft Delete) ומאפשר שחזור קל ומהיר או מחיקה לצמיתות.
* **אימות גוגל (Firebase Auth):** גישה לנתונים מותנית בהתחברות מאובטחת דרך חשבון Google, כאשר הנתונים בענן מבודדים ברמת ה-UID של המשתמש.

## 📁 מבנה הקבצים בפרויקט

| קובץ | תפקיד מהותי |
| :--- | :--- |
| `index.html` | קובץ הליבה של ה-SPA. מכיל את כל מבנה ה-HTML, עיצוב ה-CSS המוטמע (לביצועים מקסימליים), ולוגיקת ה-JavaScript (Vanilla) המלאה המנהלת את ה-State, ה-UI והתקשורת. |
| `manifest.json` | קובץ הגדרות ה-PWA. מגדיר את צבעי המערכת, תצוגה עצמאית (Standalone), אייקונים, ותצורת ההתקנה כאפליקציית שולחן עבודה / מובייל נטיבית. |
| `sw.js` | קובץ ה-Service Worker. אחראי על יירוט בקשות רשת (Fetch Interception), ניהול אסטרטגיית Network-First, ועדכון גרסאות חלק למשתמש הקצה. |
| `icon.png` | האייקון הרשמי של המערכת המוצג במסכי הבית ובשורת המשימות. |

## 🚀 הפעלה והרצה מקומית

המערכת בנויה בארכיטקטורת Serverless / Client-Side לחלוטין ולכן אינה דורשת שרת Backend (כגון PHP או Node.js) כדי לפעול. 

ניתן להריץ אותה בקלות מכל שרת HTTP פשוט לצורכי פיתוח:

> **באמצעות שרת מובנה של Python:**
> `python3 -m http.server 8080`

> **או באמצעות כלי שרת של Node.js:**
> `npx serve .`

לאחר מכן, יש לגשת בדפדפן לכתובת: `http://localhost:8080`

## 🔑 התקנת מפתחות API לבינה המלאכותית

כדי שהמערכת תוכל לתקשר עם מודלי השפה (LLMs) ולסרוק תוכן, יש לספק לה מפתח API. הפעולה נדרשת פעם אחת בלבד:

1. צור מפתח חינמי באחד מהשירותים הנתמכים:
   - **Google Gemini API (המומלץ ביותר):** [Google AI Studio](https://aistudio.google.com/app/apikey)
   - **Groq API (מהירות מקסימלית):** [Groq Console](https://console.groq.com/keys)
   - **OpenRouter API:** [OpenRouter Keys](https://openrouter.ai/keys)
2. היכנס לאפליקציה המותקנת ופתח את תפריט **✨ AI**.
3. לחץ על **⚙️ הגדרות AI ומפתחות**.
4. הדבק את המפתח שקיבלת בתיבה המתאימה, בחר את המנוע המועדף לסריקה ולחץ שמור. (המפתח מקודד ונשמר ברמת המשתמש).

## 💻 מחסנית טכנולוגית (Tech Stack)

* **Frontend Engine:** Vanilla JavaScript (ES6+), HTML5
* **Styling:** Custom CSS Variables, Animations, Responsive Grid/Flexbox, Glassmorphism
* **Backend & Cloud:** Firebase V10 SDK (Firestore Database, Google Authentication)
* **Web APIs:** Progressive Web App (PWA), Service Workers, IndexedDB, Web Share API, Clipboard API, ResizeObserver
* **AI & Fetching:** REST API Integration (Google Generative Language, Groq OpenAI-compatible, OpenRouter), CORS Proxies
